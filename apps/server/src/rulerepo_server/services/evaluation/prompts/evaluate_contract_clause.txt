## Instructions
1. Analyze whether the contract clause violates, complies with, or requires further review under the given rule.
2. Consider the clause type, governing law, and counterparty context when evaluating.
3. For MUST rules, a missing required element constitutes a violation.
4. For MUST_NOT rules, the presence of the prohibited element constitutes a violation.
5. If the clause is ambiguous or could be interpreted multiple ways, return NEEDS_CONFIRMATION.

Return your assessment as JSON with the following structure:
- verdict: "ALLOW" | "DENY" | "NEEDS_CONFIRMATION" | "ALLOW_WITH_CONDITIONS" | "REQUIRES_DISCLOSURE"
- confidence: a number between 0 and 1
- reasoning: a brief explanation of your assessment
- issue_description: if DENY, describe the specific clause problem
- fix_suggestion: if DENY, suggest revised clause language

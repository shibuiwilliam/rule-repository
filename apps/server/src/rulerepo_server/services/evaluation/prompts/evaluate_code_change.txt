Rules:
- If the code change is completely unrelated to this rule, return ALLOW with confidence 1.0.
- If the rule's preconditions are not met (e.g., the rule applies to "API endpoints" but the change is in a test file), return ALLOW.
- If any of the rule's exceptions apply (e.g., "except for internal-only APIs" and the change is to an internal API), return ALLOW.
- DENY only when the violation is clear and specific. If you are unsure, return NEEDS_CONFIRMATION.
- For SHOULD rules, prefer NEEDS_CONFIRMATION over DENY unless the violation is egregious.
- fix_suggestion must be a concrete, actionable instruction (not "consider adding..." but "add X at line Y").
- For each violation, provide at least one remediation with exact file path and line numbers.
- Set auto_applicable=true only for SHOULD-level rules where the fix is unambiguous and low-risk.
